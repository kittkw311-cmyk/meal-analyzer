# UI/UX デザインシステム仕様書 (ダイエット記録アプリ・総合タブ特化版)

本書は、Next.js および既存UIライブラリ（主に shadcn/ui や Radix UI を想定）における視覚的階層、カラーマテリアル、および実装ルールを定義する。AIアシスタント（Codex/LLMなど）は、「総合タブ（ダッシュボード）」のコンポーネントやレイアウトコードを生成・修正する際、以下のトークン、制約、および禁止事項に厳格に従うこと。

---

## 1. デザインコンセプト ＆ トーン
*   **世界観:** 先進的、知的、洗練されたテック感。毎日の自己管理モチベーションを高めるインテリジェントな画面。
*   **テーマ:** 深みのあるティール（緑・青系）と鮮烈なネオンカラーを組み合わせたダークモード（Dark Theme）。単純な黒（#000000）や白（#ffffff）の多用は避け、ダークネイビーをキャンバスとする。

---

## 2. レイアウト ＆ スタイリングの基本規則

AIはコンポーネントを実装する際、以下の数値制約を絶対厳守すること。
*   **基本グリッド（Base Spacing）:** すべての余白（Margin, Padding）や要素の間隔は **`8px` グリッド**（Tailwindの `p-2(8px)`, `p-4(16px)`, `gap-4(16px)`, `p-6(24px)` など）をベースに設計すること。
*   **角丸（Border Radius）:** 先進的でソリッドな印象を与えるため、**角丸は `4px`（控えめな指定）**とする。Tailwindの `rounded-sm(2px)` または `rounded(4px)` を使用し、それ以上の丸み（`rounded-lg` や `rounded-full` など）は原則禁止とする。
*   **コンテンツ最大幅（Max Width）:** モバイル端末での閲覧やスタイリッシュな縦長UIを担保するため、アプリ全体のコンテンツ幅は **`max-w-[412px]`** に制限し、画面中央に配置（`mx-auto`）すること。

---

## 3. タイポグラフィ（フォント指定）

*   **基本フォント（Base Font-Family）:** **`'Zen Kaku Gothic New', sans-serif`** を適用。
*   **見出しフォント（Heading Font-Family）:** ダッシュボードの見出し、タイトル、および主要な数値（体重やカロリーなど）には **`'Outfit', sans-serif`** を適用し、洗練された海外テック感・幾何学的な美しさを表現すること。
*   **フォント指定の補足:** データの数値が縦に並ぶ場所など、ブレを完全に防ぎたい場合は等幅（monospace）表現をOutfitと組み合わせること。

---

## 4. カラーシステム（デザイントークン設定）

UIライブラリが読み込む `globals.css` (Tailwind CSS 形式) の設定値を以下のように上書き・参照すること。

```css
@layer base {
  .dark {
    /* ベースカラー（70%）: 画面の骨組み */
    --background: 208 39% 11%;    /* #111d27 (ディープネイビー・全体の基本背景) */
    --card: 205 43% 15%;          /* #162a37 (ダッシュボード各カード・コンテナ背景) */
    --popover: 205 43% 15%;       /* #162a37 */
    --border: 205 39% 23%;        /* #233e50 (うっすら見える境界線) */
    --input: 205 39% 23%;         /* #233e50 */

    /* メインカラー（25%）: ブランドアイデンティティ */
    --primary: 169 62% 26%;       /* #196a5b (ディープエメラルドグリーン) */
    --primary-foreground: 167 22% 96%; /* #f0f7f6 (読みやすい文字色) */

    /* アクセントカラー（5%）: 視線を奪う最重要要素 */
    --accent: 185 100% 50%;       /* #00e5ff (ネオンシアン・サイバーブルー) */
    --accent-foreground: 208 39% 11%; /* #111d27 (コントラスト用ダークネイビー) */

    /* タイポグラフィ・その他 */
    --foreground: 167 22% 96%;    /* #f0f7f6 (オフホワイト・重要文字やメインの数値) */
    --muted: 171 12% 61%;         /* #8fa7a3 (くすんだセージグレー・補足文字や単位) */
    --muted-foreground: 171 12% 61%;

    --ring: 185 100% 50%;         /* フォーカス時はネオンシアンを光らせる */

    /* 📊 チャート・グラフ用カラー (shadcn/ui Chart対応) */
    --chart-1: 185 100% 50%;     /* #00e5ff (ネオンシアン: 体重ライン / 目標値) */
    --chart-2: 169 62% 26%;     /* #196a5b (ディープエメラルド: 体脂肪率ライン / タンパク質) */
    --chart-3: 35 100% 52%;      /* #ff9900 (アンバーオレンジ: 脂質) */
    --chart-4: 205 39% 23%;      /* #233e50 (ダークグレーネイビー: 炭水化物・未達成エリア) */
  }
}
```

## 5. 総合タブ（ダッシュボード）構成・実装ルール

Codexは、最大幅 `max-w-[800px]` の中で「総合タブ」を構築する際、必ず以下のコンポーネント構造とスタイリングを再現すること。

### 5.1 体組成サマリー（数値カード群）
*   **現在の体重カード・目標までの残りカード:**
    *   背景は `--card` (`#162a37`)、角丸は厳格に `rounded` (4px) とする。
    *   見出しおよびメインの数値（例：`68.5`）には `font-heading` (`Outfit`) を適用し、大きく太字で表示。単位（`kg`）やラベルは `font-sans` (`Zen Kaku Gothic New`) かつ `--muted` で一回り小さく表示する。
    *   「前日比」や目標達成へのポジティブな進捗数値には `--accent` (`#00e5ff`) を適用すること。

### 5.2 体組成トレンド（折れ線グラフ）
*   **体重・体脂肪率の推移グラフ:**
    *   shadcn/ui の `<ResponsiveContainer>` と `<LineChart>` をベースにする。
    *   体重の折れ線（Line）の `stroke` には `--chart-1`、体脂肪率は `--chart-2` を指定。
    *   グリッド線（`<CartesianGrid>`）の `stroke` には必ず `--border` (`#233e50`) を指定し、不透明度を低く（`opacity={0.3}`）設定すること。
    *   グラフ内のX軸・Y軸のフォントも `font-heading` (`Outfit`) で統一する。

### 5.3 食事・マクロ栄養素（カロリー ＆ PFC進捗）
*   **総摂取カロリー進捗バー:**
    *   shadcn/ui の `<Progress>` コンポーネントを使用。角丸は `rounded-[4px]` に上書きする。
    *   進捗バーのインジケータ（満たされている部分）には `--accent` または `--primary` を適用する。
*   **PFCバランス（タンパク質・脂質・炭水化物）:**
    *   **P（タンパク質）**: `--chart-2` (ディープエメラルド)
    *   **F（脂質）**: `--chart-3` (アンバーオレンジ)
    *   **C（炭水化物）**: `--chart-1` (ネオンシアン) または `--muted`

## 6. デザインアンチパターン（禁止事項）

AIアシスタントは、コード生成時に以下の「世界観を壊す実装」を絶対に避けること。

### 6.1 原色や一般的なカラーの単体使用（禁止）
*   **純粋な黒・白の禁止:** 背景に `bg-black (#000000)`、文字に `text-white (#ffffff)` をそのまま使用しない。必ず `--background` や `--foreground` の変数を経由させること。
*   **標準的な赤・緑・青の禁止:** アラートやグラフだからといって、Tailwind標準のビビッドな赤（`bg-red-500`）や緑（`bg-green-500`）をそのまま使わない。必ず定義されたデザイントークン（`--accent` や `--chart-3`）のトーンに合わせること。

### 6.2 ダイエットアプリ特有の「ほのぼの感」の排除
*   **パステルカラー・大きな丸みの禁止:** 一般的な健康管理アプリに多い「パステルピンク」「ライムグリーン」などの優しい色合いや、ポップなアイコン（可愛いイラスト等）は使用しない。角丸は必ず `4px` 以下のシャープな形状を維持すること。
*   **指定外フォントの混入禁止:** 游ゴシックや明朝体、丸ゴシックなどが絶対に混ざらないよう、CSSおよびTailwindのフォント指定を徹底すること。

### 6.3 スタイリッシュさを損なう不適切なレイアウト
*   **太すぎる区切り線:** 境界線（Border）に必要以上の太さ（`border-2` 以上）を持たせたり、明るい色を適用しない。区切り線は `--border` を使い、基本は `border-[1px]` でうっすらと見せる。
*   **過剰な発光（グロー）:** 未来感を出すためのグロー効果（`shadow-[rgba(...)]`）を、すべてのカードやテキストに乱用しない。発光は「目標達成時」や「最重要ハイライト」の1〜2箇所に限定する。
